# Doors as lenses

The core design concept, decided 2026-06-11. Martin's pushback when asked which
categories the site should be "organized around": *"the point is that it won't?
… it's organized around the fact that I'm doing many different things? INTP /
hyper-fixations kind of thing maybe."* (lightly paraphrased)

Therefore:
- **One content pool.** Every item (post, project, video, paper) carries facet tags
  (`dev`, `physics`, `fiction`, `music`, `ideas`…). Items can carry several facets.
- **Doors = pre-applied filters + tailored intro.** The Software door shows the CV
  pitch and dev-tagged content; the Physics door shows the preprint/ORCID and
  physics-tagged posts. Same pool underneath.
- **Always an escape hatch** — every filtered view links to "everything" (Martin
  explicitly wanted "the option to see them all").
- **No forced gate.** Doors live on a normal homepage; deep links and SEO work.

Working door set: Software · Physics · Book · Music · Everything. Wording and count
still open (PLAN.md open question #5) — doors are cheap, they're just filters.
